# Конгруэнтные генераторы - генератор Парка-Миллера

Генератор Парка-Миллера - это линейный генератор (конгруэтный)

Принцип работы:

Генератор вычисляет очередное число по рекурректной формуле. После каждый следующий элемент последовательности получается путем умножения предыдущего на фиксированный множитель и каждый раз берет последний бит этого числа..


Xₙ₊₁ = (a × Xₙ) mod m

## 1. Генерация бита
```
let q: Int64 = 127773  // q = m / a
let r: Int64 = 2836    // r = m % a
```
Прямое умножение на 32-битных целых числах может вызвать переполнение.
Поэтому используется оптимизация Шраге.

q и r - предвычисленные константы для метода

q = floor(m / a) = 127773

r = m % a = 2836

## 2. Инициализация
```
init(seed: Int64) {
    self.state = seed % m
    if self.state <= 0 { self.state = 1 }
}
```
Начальное значение (seed) необходимо для старта рекурентного процесса. По сути оно задает стартовую точку в последовательности псевдослучайных чисел, гарантируя что при одном и том же начальном значении мы получим повторяемый результат.

Инициализирует генератор начальным значением (seed), после приводим seed к диапазону, а поскольку ноль недопустим, поэтому заменяется на 1.

## 3. Основные вычисления
```
let hi = state / q      // целая часть от деления
let lo = state % q      // остаток
var t = a * lo - r * hi // вычисление без переполнения
if t <= 0 { t += m }    // коррекция отрицательных значений
state = t               // новое состояние
```
Сначала мы разделяем на два состояния на старшую (хранит сколько раз пройдет q) и младшую часть (остаток от деления).

Далее мы производим вычисление (t) без переполнения

Если результат получится отрицательным, то мы прибавляем m. Это необходимо для того, чтобы вернуть число в положительный диапазон.
## 4. Возврат бита
```
return UInt8(state & 1)  // младший бит состояния
``` 
Для получения псевдослучайного бита берут младший бит состояния

Потому что в данном генераторе младшие биты чередуются, что дает быстрый и простой источник битовой энтропии.

# Алгоритм BBS

Этот алгоритм основывается на проблеме факторизации больших чисел. Он генерирует биты путем возведения текущего числа в квадрат и  взятие остатка от деления на модуль. Каждый раз берется последний бит результата.

Выбираем два простых числа, которые при делении на 4 дают остаток 3. (Чем сложнее разложить modulus на множители, тем надежнее генератор)
```
let p: UInt64 = 383     // простое число
let q: UInt64 = 503     // простое число
modulus = p * q         // 383 * 503 = 192,649
```
modulus = количество комбинаций

seed = начальная комбинация

## 1. Инициализация состояния
```
state = seed % modulus //вычисляем по модулю
if state == 0 { state = 1 } // 0 нельзя иначе зацикливание
```
Берем начальное число (seed) которое взаимнопростое с n при этом не кратно p и q.

Состояние не должно быть равно 0 иначе генератор зациклится и будет всегда возвращать 0
## 2. Инициализация состояния
```
func nextBit() -> UInt8 {
        state = (state * state) % modulus // преобразовывает
        return UInt8(state & 1) // извлекает случайный бить
    }
```
На каждом шаге мы выполняем две операции. Текущее состояние возводится в квадрат и берется остаток от деления на модуль state.

После мы извлекаем из полученого числа самый младший бит. (как правило это последняя цифра в двоичной записи)

# FIPS 186 

FIPS 186 - это детерминированный псевдослучайный генератор. 

На вход подается значение seed. Мы приводим его к 20 байтному значению. Если оно короче, то дополняем нулями слева. Если длинее, то берем первые 20 байт. Полученные 20 байт становятся текущим состоянием x.
```
init(seed: Data) {
    var seedData = seed
    if seedData.count < 20 {
        seedData = Data(repeating: 0, count: 20 - seedData.count) + seedData
    }
    self.x = seedData.prefix(20)
}
```

Цикл будет повторяться пока не наберет нужное кол-во бит.

Сначала вычисляет хеш. Хеш перемешивает биты состояния и создает псевдослучайный выход.

```
    let xval = currentX
var digest = Data(count: Int(CC_SHA1_DIGEST_LENGTH))
digest.withUnsafeMutableBytes { dPtr in
    xval.withUnsafeBytes { xPtr in
        _ = CC_SHA1(
            xPtr.baseAddress,
            CC_LONG(xval.count),
            dPtr.baseAddress!
        )
    }
}
```
Далее обновляет состояние, чтобы следующая итерация дала новые биты. Прибавлять 1 необходимо, чтобы гарантировать что цикл не зациклится.

После мы обновляем состояние (складываем и отбрасываем переполнение)
```
    currentX = addMod160(addMod160(currentX, digest), makeOne())
    private func addMod160(_ a: Data, _ b: Data) -> Data { ... }

private func makeOne() -> Data {
    var d = Data(count: 20)
    d[19] = 0x01
    return d
}
```
Из полученного состояния 20 байт берем все 160 бит по порядку (младший бит каждого байта идет первым)

for byte in digest {

```

```
