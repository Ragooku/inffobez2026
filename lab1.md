# Частоточный тест

Частоточный тест проверяет соотношение нулей и единиц во всей последовательности.

1 -> +1; 

0 -> -1;

Если единиц больше, то сумма положительная

Если нулей больше, то сумма отрицательная

Если поровну, то сумма близка к нулю.

Тест пройден если 0 и 1 равномерно

Тест не пройден еслии явный перекос 0 или 1

```
var sum = 0.0
        
for bit in bitSequence {
    sum += (bit == 1) ? 1.0 : -1.0
}
```
Далее сравниваем с порогом 
```
let threshold = 1.82138636
return (statistic <= threshold, statistic)
```
Если статистика меньше или равна порогу → тест пройден

Если статистика больше порога → тест не пройден

Порог 1.82138636 — это критическое значение статистики начение соответствует квантилю нормального распределения

# Тест на последовательность

Данный тест проверяет насколько чередуюутся 0 и 1 в последовательности. Он проверяет порядок.

## 1. Вычисляем долю единиц

```
// проходит по всем элементам массива и суммирует их
let ones = bitSequence.reduce(0) { $0 + Int($1) } 
// это доля единиц (от 0 до 1)
// Чем ближе pi к 0.5, тем больше ожидаемое число серий.
let pi = Double(ones) / n
```
## 2. Подсчет серий

Мы идем с 1, переходим ко 2 элементу и сравниваем текущий бит с предыдущим, если они разные, то начинается новая серия, если нет увеличивает счетчик.
```
var runs = 1
for i in 1..<bitSequence.count {
    if bitSequence[i] != bitSequence[i - 1] {
        runs += 1
    }
}
```
## 3. Вычисление статистики

numerator - Модуль разности между реальным числом серий

denominator - Нормирующий множитель, связанный со стандартным отклонением распределения числа серий.

statistic - показывает, насколько реальное значение отклоняется от ожидаемого в единицах стандартного отклонения.
```
let numerator = abs(Double(vn) - 2.0 * n * pi * (1.0 - pi))
let denominator = 2.0 * sqrt(2.0 * n) * pi * (1.0 - pi)
let statistic = numerator / denominator
```
## 4. Интерпретация результата

Порог 1.82138636 — это критическое значение статистики начение соответствует квантилю нормального распределения

Если разница между реальным и ожидаемым числом серий статистически незначима -> тест ПРОЙДЕН 

Если разница слишком велика -> тест НЕ ПРОЙДЕН

Сравниваем: 1.414 ≤ 1.821 -> тест пройден

```
let threshold = 1.82138636
return (statistic <= threshold, vn, statistic)
```

# Тест на произвольные отклонения

Тест на произвольные отклонения - статистический тест для проверки случайности последовательности чисел. 

Цель теста — определить, отличается ли число посещений определённого состояния внутри цикла от аналогичного числа в случае абсолютно случайной входной последовательности.

Суть теста заключается в подсчёте числа циклов, имеющих строго определённое количество посещений при произвольном обходе кумулятивной суммы.

## 1. Построение кумулятивных сумм (случайное блуждание)
Случайное блуждание - последовательность битов превращается в траекторию движения. Тест анализирует, сколько раз вы посещаете каждую позицию на этой прямой.

```
// Кумулятивные суммы 0 -> -1 1->+1
var partialSums = [Int]()
partialSins.reserveCapacity(n + 2)
partialSums.append(0)

var current = 0
for bit in bitSequence {
    current += (bit == 1) ? 1 : -1
    partialSums.append(current)
}
partialSums.append(0)  // S'_{n+1} = 0
```
Начинаем с позиции 0 и для каждого бита делаем шаг 1 -> +1 (вправо) 0 -> -1 (влево)

После запоминаем все промежуточные позиции и в конце добавляем 0

## 2. Определение циклов
Это путь от одного возврата до следующего, а так же кол-во возвратов в 0 минус 1.

```
// Количество возвращений в 0 (циклов)
let zerosCount = partialSums.filter { $0 == 0 }.count
let L = zerosCount - 1  // количество циклов
```
Если нет циклов, то тест провести нельзя
```
if L <= 0 {
    return (false, 18, Set(-9...9).subtracting([0]))
}
// Возвращаем: не пройден, 18 ошибок, все состояния от -9 до -1 и от 1 до 9
```
## 3. Подсчет посещений состояний
```
// Подсчёт состояний
var visits = [Int: Int]()
for j in -9...9 where j != 0 {
    visits[j] = 0
}

for s in partialSums {
    if let count = visits[s], s != 0 {
        visits[s] = count + 1
    } else if s != 0 && abs(s) <= 9 {
        visits[s] = 1
    }
}
```

Инициализируем счетчик для каждого состояния от -9 до -1 и от 1 до 9

Проходим по всем позициям в partialSums

Если позиция уже есть в словаре и это не 0 -> увеличиваем счетчик

Если позиции нет в словаре, но |s| ≤ 9 и s ≠ 0 → создаем запись со значением 1

## 4. Вычисление статистики для каждого состояния
```
var failed = Set<Int>()
let threshold = 1.82138636

for j in -9...9 where j != 0 {
    guard let xi = visits[j] else { continue }
    
    let denom = sqrt(2.0 * Double(L) * (4.0 * Double(abs(j)) - 2.0))
    if denom <= 0 { continue }
    
    let yj = abs(Double(xi) - Double(L)) / denom
    
    if yj > threshold {
        failed.insert(j)
    }
}
```

## 5. Интерпретация результатов
```
return (failed.isEmpty, failed.count, failed)
```
passed: true, если нет ни одного проваленного состояния

failedCount: количество состояний, где статистика превысила порог

failedStates: множество проваленных состояний (например, {2, -3, 5})
